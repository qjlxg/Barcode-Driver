            <section id="about">
                <h2>🎤 О проекте Twitch TTS</h2>
                <p><strong>Twitch TTS (Text-to-Speech)</strong> — это клиентское веб-приложение (SPA), разработанное для стримеров. Оно позволяет в реальном времени интегрировать текстовые сообщения из чата Twitch в голосовую озвучку прямо на трансляции, используя гибкую систему кастомных голосов и прав доступа.</p>
                <div class="info-block">
                    <p>Интерфейс выполнен в фирменной темной палитре Twitch для бесшовной интеграции в рабочее окружение контент-мейкера.</p>
                </div>
            </section>

            <section id="architecture">
                <h2>🛠️ Архитектура проекта</h2>
                <p>Приложение представляет собой легковесный автономный скрипт, не требующий сложного серверного деплоя:</p>
                <ul>
                    <li><strong>Twitch IRC Интеграция:</strong> Использует библиотеку <code>Comfy.js</code> (подключена через CDN). Это позволяет слушать события чата без необходимости прохождения OAuth-авторизации со стороны стримера — достаточно указать имя канала.</li>
                    <li><strong>Бизнес-логика:</strong> Чистый ванильный JavaScript (ES6+), управляющий очередями воспроизведения, механикой повторных запросов, обработкой ошибок сети и событий элементов HTML5 Audio.</li>
                    <li><strong>UI/Интерфейс:</strong> Трехколоночная структура (Голоса, Команды управления, Активность пользователей), построенная на семантичном HTML и кастомных CSS стилях без применения тяжелых UI-фреймворков.</li>
                </ul>
            </section>

            <section id="mod-commands">
                <h2>🛡️ Административные команды управления</h2>
                <p>Модераторы и владелец канала (Broadcaster) обладают расширенным синтаксисом для динамической настройки поведения системы:</p>
                
                <h3>Управление режимами автоматического чтения чата:</h3>
                <ul>
                    <li><code>!tts on</code> / <code>!tts off</code> — Включение или отключение глобального перехвата и озвучивания **всех** рядовых сообщений чата.</li>
                    <li><code>!tts on [username]</code> / <code>!tts off [username]</code> — Таргетированное включение/исключение автоматической озвучки для конкретного зрителя.</li>
                    <li><code>!tts random [секунды]</code> — Активация режима «Случайный спикер». Система будет брать последнее живое сообщение из чата и зачитывать его раз в указанный интервал времени со случайным голосом. Для выключения используется <code>!tts random off</code>.</li>
                    <li><code>!tts clear</code> — Экстренная полная очистка текущей очереди и мгновенное прерывание проигрываемого в данный момент аудиопотока.</li>
                </ul>

                <h3>Менеджмент прав доступа (Синтаксис <code>!open</code> / <code>!close</code>):</h3>
                <ul>
                    <li><code>!open all</code> / <code>!close all</code> — Выдача или отзыв прав на использование ручных команд <code>!tts</code> и <code>!s</code> для абсолютно всех зрителей.</li>
                    <li><code>!open vip</code> / <code>!close vip</code> — Открытие или закрытие функционала эксклюзивно для аудитории со статусом VIP.</li>
                    <li><code>!open [username]</code> / <code>!close [username]</code> — Индивидуальный белый или черный список для конкретного аккаукта.</li>
                </ul>

                <h3>Административное назначение голосов:</h3>
                <ul>
                    <li><code>!s [username] [voice_id]</code> — Принудительное назначение или изменение голоса выбранному пользователю модератором.</li>
                </ul>
            </section>

            <section id="queue-logic">
                <h2>🧠 Механика управления очередью (PlayNext)</h2>
                <p>Воспроизведение базируется на изолированном асинхронном цикле во избежание наложения звуковых дорожек друг на друга:</p>
                <pre><code>async function playNext() {
  if (playing || queue.length === 0) return;
  playing = true;
  
  const item = queue[0];
  // Формирование URLSearchParams, валидация токенов...
  
  try {
    const res = await fetchWithRetry(...);
    let blob = await res.blob();
    const url = URL.createObjectURL(blob);
    const audio = new Audio(url);
    
    audio.onended = () => {
      playing = false;
      URL.revokeObjectURL(url);
      setTimeout(() => playNext(), TTS_GAP_BETWEEN_MS); // Пауза 4 секунды
    };
    
    await audio.play();
  } catch(err) {
    playing = false;
    setTimeout(() => playNext(), 1200);
  }
}</code></pre>
                <p>Каждому запуску функции присваивается уникальный инкрементируемый <code>playbackToken</code>. Это защищает систему от дублирования звука в случаях, когда модератор прожал очистку очереди во время ожидания сетевого ответа от бэкенда.</p>
            </section>

            <section id="retry-logic">
                <h2>📡 Защита от сбоев и Retry-политика</h2>
                <p>Приложение спроектировано с учетом нестабильности сетевых маршрутов и работы под высокими нагрузками:</p>
                <ul>
                    <li><strong>Обработка HTTP HTTP 429 (Too Many Requests):</strong> При возникновении лимитов на стороне сервера генерации, функция <code>fetchWithRetry</code> пытается проанализировать HTTP-заголовок <code>Retry-After</code>. Если заголовка нет, запускается алгоритм экспоненциального отката (увеличение задержки в геом. прогрессии <code>2^(attempt-1)</code>) с информированием стримера на панели статусов.</li>
                    <li><strong>Фильтрация спама ссылками:</strong> На этапе перехвата события <code>onChat</code> строка очищается регулярными выражениями от любых URL-адресов. Это защищает стрим от зачитывания роботом длинных бессмысленных веб-ссылок.</li>
                </ul>
            </section>

            <section id="localstorage">
                <h2>💾 Экосистема локального хранения (LocalStorage)</h2>
                <p>Для сохранения конфигурации при перезапуске вкладки браузера (или OBS сцены) задействован механизм <code>localStorage</code>:</p>
                <ul>
                    <li><code>ttsChannel</code> — Хранит текстовое имя последнего подключенного Twitch-канала.</li>
                    <li><code>ttsAutoPlayState</code> — Сериализованный в JSON объект, содержащий глобальный статус флага Авто-TTS и списки исключений (массивы пользователей с персонально включенной/выключенной озвучкой).</li>
                    <li><code>ttsPerUserSpeakers</code> — Таблица маппинга вида <code>{"логин_пользователя": "идентификатор_голоса"}</code>, к которой обращается система для поиска голоса при генерации.</li>
                </ul>
